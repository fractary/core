# Standards Documentation Standards

## Required Conventions

### 1. Clear Scope
- ALWAYS define what standard covers
- ALWAYS specify applicability (who/what/when)
- ALWAYS include rationale for standard
- ALWAYS version standards

### 2. Requirements
- ALWAYS distinguish MUST vs. SHOULD requirements
- ALWAYS provide examples of compliance
- ALWAYS include non-examples (anti-patterns)
- ALWAYS document exceptions

### 3. Validation
- ALWAYS specify how compliance is checked
- ALWAYS include validation tools if applicable
- ALWAYS document validation frequency
- ALWAYS track compliance metrics

### 4. Maintenance
- ALWAYS review standards periodically
- ALWAYS track standard adoption
- ALWAYS document changes in CHANGELOG
- ALWAYS communicate updates to stakeholders

## Best Practices

- Keep standards concise and actionable
- Link to examples and templates
- Provide rationale for each requirement
- Include compliance checklist
- Make standards discoverable and searchable
