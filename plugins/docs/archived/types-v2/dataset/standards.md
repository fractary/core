# Dataset Documentation Standards

## Required Conventions

### 1. Comprehensive Coverage
- ALWAYS document schema definition
- ALWAYS include metadata (size, update frequency, owner)
- ALWAYS document access patterns and query examples
- ALWAYS include data quality metrics

### 2. Schema Documentation
- ALWAYS provide complete field definitions
- ALWAYS document data types and constraints
- ALWAYS include field descriptions and examples
- NEVER omit required fields

### 3. Data Quality
- ALWAYS document data quality standards
- ALWAYS include validation rules
- ALWAYS document known quality issues
- ALWAYS provide quality metrics

### 4. Access & Usage
- ALWAYS document how to access the dataset
- ALWAYS provide query examples
- ALWAYS document access permissions
- ALWAYS include performance considerations

### 5. Governance
- ALWAYS document data ownership
- ALWAYS include retention policies
- ALWAYS document compliance requirements (GDPR, etc.)
- ALWAYS maintain audit trail

## Best Practices

- Keep schema definitions up to date
- Document all transformations applied to data
- Provide sample data and example queries
- Link to related datasets and dependencies
- Document data lineage and sources
