# Infrastructure Documentation Standards

## Required Conventions

### 1. Resource Inventory
- ALWAYS document all infrastructure resources
- ALWAYS include ARNs and console links
- ALWAYS maintain dependency graph
- ALWAYS document resource outputs

### 2. Operational Procedures
- ALWAYS document step-by-step procedures
- ALWAYS include prerequisites and validation steps
- ALWAYS provide rollback procedures
- ALWAYS estimate time and downtime

### 3. Monitoring & Alerting
- ALWAYS document key metrics and thresholds
- ALWAYS configure alarms for critical metrics
- ALWAYS link to dashboards
- ALWAYS specify log locations

### 4. Disaster Recovery
- ALWAYS document RTO and RPO
- ALWAYS include backup strategy
- ALWAYS provide recovery procedures
- ALWAYS test recovery regularly

### 5. Security & Compliance
- ALWAYS document access controls
- ALWAYS include compliance requirements
- ALWAYS maintain security configurations
- ALWAYS track security issues

## Best Practices

- Keep architecture decisions documented with rationale
- Maintain cost optimization opportunities
- Document troubleshooting guides for common issues
- Include contacts for on-call and escalation
- Keep documentation in sync with infrastructure changes
