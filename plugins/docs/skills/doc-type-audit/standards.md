# Audit Documentation Standards

## Required Conventions

### 1. Executive Summary
- ALWAYS include overall status and score
- ALWAYS provide status breakdown
- ALWAYS document audit duration
- ALWAYS specify compliance percentage

### 2. Findings
- ALWAYS categorize by severity (critical, error, warning, info)
- ALWAYS include remediation steps
- ALWAYS link findings to affected resources
- ALWAYS prioritize findings

### 3. Metrics
- ALWAYS document resources/items audited
- ALWAYS include coverage percentage
- ALWAYS track quality scores
- ALWAYS provide domain-specific metrics

### 4. Recommendations
- ALWAYS prioritize recommendations
- ALWAYS include effort estimates
- ALWAYS document impact and rationale
- ALWAYS link to related findings

### 5. Remediation
- ALWAYS provide actionable remediation plans
- ALWAYS track remediation progress
- ALWAYS link to tracking issues
- ALWAYS document auto-fix results

## Best Practices

- Generate audit reports regularly (daily/weekly)
- Track audit trends over time
- Link audits to remediation specifications
- Document domain-specific extensions
- Maintain historical audit data for analysis
